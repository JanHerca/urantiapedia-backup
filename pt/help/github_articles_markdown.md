Aqui é descrito como obter os artigos dos leitores relacionados ao _O Livro de Urântia_, e que foram catalogados seguindo a tarefa [Creating an article catalog](/pt/help/github_articles_catalog).

## Processo

Tudo o que foi dito sobre livros em [Obter livros “fonte”](/pt/help/github_sourcebooks_markdown) também se aplica aqui. Os artigos podem ser encontrados na forma de uma página web que já os hospeda, como PDF sem texto, ou como PDF com texto. Eles também podem ser individualmente ou como parte de uma revista ou publicação conjunta com outros artigos. De qualquer forma, o primeiro passo é converter essa revista para o formato Markdown.

Esta tarefa, realizada com software diferente dependendo do caso, já foi feita em parte com uma boa coleção de artigos, que estão disponíveis no GitHub em `/input/markdown/en/Innerface` e `/input/markdown/en/Herald` para duas publicações bem conhecidas relacionadas ao _O Livro de Urântia_, o *Innerface International* e o *Fellowship Herald*. O catálogo de artigos já está construído para essas publicações aqui: [Artigos planejados](/en/index/articles).

A tarefa agora é utilizar o conteúdo das pastas mencionadas e construir os artigos individuais, pois nessas pastas existem revistas completas.

### Estrutura de artigos em Urantiapedia

Se você for um “editor chefe” e estiver usando a plataforma GitHub, os artigos devem ser salvos nestas pastas do projeto GitHub: `/output/wikijs/en/article` para artigos em inglês, `/output/wikijs/es/article` para artigos em espanhol, etc. Dentro dessas pastas criaremos outra, caso não exista, com o nome do autor (com espaços em branco substituídos por sublinhados). Por fim, dentro da pasta do autor, criaremos um arquivo Markdown ao qual daremos um nome exclusivo seguindo este critério:
  - O nome usará o título do artigo, mas o mais abreviado possível.
  - O nome estará sempre em inglês, portanto, se o artigo original não estiver em inglês, o título será traduzido.
  - Os espaços em branco são substituídos por sublinhados.
  - Caracteres especiais como dois pontos ou exclamações são removidos.
  - Por exemplo, se o título de um artigo for *Time for More Change? Símbolos, Cultos e O Livro de Urântia* um possível nome de arquivo seria `Time_for_More_Change_Symbols_Cults.md`. Se o título for *Jerusalén en tiempos de Jesús* (espanhol), o arquivo seria `Jerusalem_in_Jesus_times.md`.

Se você é um “Assistente Editor” e não vai usar a plataforma GitHub para enviar arquivos, basta criar uma pasta no seu PC para deixar os artigos que você criar. Então, quando você enviar seu trabalho para um Administrador, ele cuidará de criar as pastas corretas.

Um exemplo de artigo finalizado pode ser visto em `/output/wikijs/en/article/Ann_Bendall/Adam_an_Eve.md`.

### Formato básico de um artigo sobre Urantiapedia

O formato a ser utilizado é o Markdown, que é extremamente simples de dar estilo em comparação com outras alternativas.

O “Editor Assistente” deve revisar o texto e adicionar o seguinte:
- Um cabeçalho em cada arquivo.
- Imediatamente abaixo do cabeçalho, indicação do autor.
- Marcas de seção dentro do artigo.
- Corrigir parágrafos.
- Negrito e itálico.
- Citações.
- Imagens.
- Referências a imagens.
- Notas de rodapé.
- Tabelas.
- Fórmulas matemáticas ou caracteres em notação científica.

Todos esses itens já foram explicados ao falar sobre como formatar capítulos de livros: [Obter livros “fonte”](/pt/help/github_sourcebooks_markdown). Todas as dicas dadas lá também se aplicam aqui no caso de artigos.

## Enviando arquivos

Se você for um “editor-chefe” usando o GitHub, poderá enviar artigos conforme detalhado em [Ajuda dos editores-chefe](/pt/help/github); se você for um “Editor Assistente”, você pode simplesmente enviar os arquivos Markdown para o e-mail de qualquer Administrador ou para urantiapedia@gmail.com.
